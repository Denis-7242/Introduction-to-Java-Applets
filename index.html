‚òï What is a Java Applet?

Definition: A Java applet is a small application written in Java that runs within a web browser or applet viewer. It's designed to be embedded in HTML pages and executed on the client side.

Brief History

  • 1995: Introduced with Java 1.0 by Sun Microsystems
  • Mid-1990s to 2000s: Peak popularity for interactive web content
  • 2017: Deprecated in Java 9
  • 2018: Removed from JDK 11 onwards

‚úì Why Use Applets?

Advantages

  • Platform independence (Write Once, Run Anywhere)
  • Rich interactive content in browsers
  • Automatic updates without reinstallation
  • Access to full Java API

Common Uses

  • Educational simulations and visualizations
  • Interactive games
  • Data visualization tools
  • Online calculators and converters

‚ö† Limitations and Decline

  • Plugin Dependency: Required Java Runtime Environment (JRE) plugin installed in browsers
  • Security Vulnerabilities: Numerous exploits made applets a security risk
  • Performance Issues: Slow startup times and resource-intensive
  • Browser Support: Major browsers gradually removed support (Chrome 2015, Firefox 2017, Edge never supported)
  • Official Deprecation: Marked deprecated in Java 9 (2017) and removed in Java 11 (2018)
  • Mobile Incompatibility: Never worked on mobile devices

</> Applet Structure Example

A basic Java applet extends the java.applet.Applet class: 

import java.applet.Applet;
import java.awt.Graphics;

public class HelloApplet extends Applet {
    
    @Override
    public void init() {
        // Initialization code
        setBackground(Color.white);
    }
    
    @Override
    public void paint(Graphics g) {
        // Drawing code
        g.drawString("Hello, World!", 50, 50);
    }
}

  • init(): Called once when applet is loaded; used for initialization
  • start(): Called when applet becomes visible; starts or resumes execution
  • paint(): Called to render graphics on screen
  • stop(): Called when applet becomes invisible; pauses execution
  • destroy(): Called once before applet is unloaded; cleanup resources

üåê Embedding Applets in HTML

Applets were embedded using the <applet> tag (now obsolete): 

<html>
<head>
    <title>My Java Applet</title>
</head>
<body>
    <h1>Welcome to My Applet</h1>
    
    <applet code="HelloApplet.class" 
            width="400" 
            height="300">
        <param name="bgcolor" value="white">
        Your browser does not support Java Applets.
    </applet>
</body>
</html>

Note: The <applet> tag was deprecated in HTML 4 and removed in HTML5.

üì¶ Types of Applets

Local Applets

Stored on the local file system. Faster to load but less portable. 

code="file:///C:/applets/MyApplet.class"

Remote Applets

Downloaded from a web server. More common for web distribution. 

codebase="http://example.com/applets/"

Signed Applets

Digitally signed with certificates to gain additional permissions beyond the security sandbox. Required user approval before running.

üîí Security & Sandbox

Applets ran in a restricted security environment called a sandbox:

Restrictions Applied:

  • Cannot read or write files on the local file system
  • Cannot execute programs on the host machine
  • Can only make network connections to the originating server
  • Cannot access system properties except approved ones
  • Cannot load native libraries or define native methods

Purpose:

The sandbox protected users from malicious code, but signed applets could request additional permissions through user-approved certificates.

üöÄ Why Applets Became Obsolete

Modern Alternatives Emerged:

  • HTML5: Native canvas element, video/audio support, and rich APIs
  • JavaScript & Frameworks: React, Angular, Vue.js for interactive UIs
  • WebAssembly: Near-native performance in browsers
  • Progressive Web Apps (PWAs): App-like experiences without plugins
  • CSS3: Advanced styling and animations without programming

Result: Modern web technologies offer better performance, security, and mobile support without requiring plugins.

üìù Summary & Takeaways

Key Points to Remember:

  • Java applets were pioneering technology for interactive web content in the 1990s-2000s
  • They required browser plugins and ran in a security sandbox
  • Security vulnerabilities and lack of mobile support led to their decline
  • Officially deprecated in 2017 and removed from Java in 2018
  • Modern web technologies (HTML5, JavaScript, WebAssembly) have completely replaced applets

Historical Significance:

While obsolete today, applets played a crucial role in demonstrating the potential of rich, interactive web applications and paved the way for modern web development.
